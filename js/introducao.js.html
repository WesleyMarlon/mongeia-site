JS: O cérebro do site

Nesse momento a área de navegação começa a ficar "complexo". Eu não vou te esconder nada. Estamos no JavaScript agora e ele com certeza não é tão fácil quanto HTML + CSS.

Eu não quero que pense que programação é algo de outro nível. Mas eu tenho que deixar claro que existem áreas da programação que não são tão fáceis e o JS é uma delas. Ele é diferente, pois ao contrário do HTML que é uma linguagem de marcação e o CSS que é uma linguagem de estilo. O JS é uma linguagem de programação.

Mas fique tranquilo que não vai ser difícil. Mas lembre-se, estamos na parte da navegação e aqui o conteúdo vai ser voltado a isso. Eu vou te ensinar tudo do JS? Sim, mas em partes. Aqui vai ser a parte de navegação. Já na parte do Backend vai ser ensinado as partes mais técnicas do JavaScript.

O que é JS?

O JavaScript é o que faz seu site deixar de ser uma revista e se transformar em um app web.

Um site funciona sem JS? Sendo honesto. Sim, mas depende. O YouTube, por exemplo, ele não funciona sem JS e isso não é um exagero. Ele NÃO FUNCIONA sem JS. O HTML roda sozinho, mas fica melhor com CSS.

Um site só se torna um "site de verdade" quando tem JavaScript nele. Ainda mais quando é programado de forma excelente. Nesse caso esqueça, não tem como um site só com HTML e CSS se comparar a um que tem um JS bem programado.

O que o JS faz na prática?

Ele não faz uma coisa e sim várias:

  • Interatividade: Abrir/fechar menu, curtir post, validar senha no formulário antes de enviar.
  • Atualizar conteúdo: Carrossel de imagens, feed do Instagram, "carregar mais". Busca em tempo real enquanto você digita. Carrinho de compras que atualiza o total na hora.
  • Reagir ao usuário: Dark mode, arrastar e soltar arquivo, arrastar card no Trello. Mouse hover com animação, scroll infinito do TikTok.

Se você juntar o JS + Node.js isso muda mais ainda. Podendo rodar o site no servidor com: login, pagar, salvar no banco de dados, mandar email, criar API. É o mesmo JS, só que em outro lugar.
