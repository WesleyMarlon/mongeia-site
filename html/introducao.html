HTML: A estrutura do site

Eu quis começar ensinando sobre Frontend. Por quê? É o mais fácil de aprender e o que mais rápido te coloca no mercado. Se você começar pelo Frontend, sua mente abre para o mercado. Porque com a trindade do Frontend: HTML, CSS e JS, você consegue fazer o site da padaria que você vai todo dia. Simples assim. Você deixa de ser um programador e começa a pensar no mercado. Começa aprendendo a vender o seu serviço.

O que é HTML?

Tudo começa pelo HTML, ele é o esqueleto do seu site.

HTML é o conteúdo do seu site. Esse texto que você está lendo é HTML.

HTML é todo conteúdo: texto, imagem, vídeo, botão. CSS é a aparência. HTML funciona sem CSS, mas fica com cara de site anos 2000. Navegador usa um estilo básico se você não escrever CSS.

Quer um site simples? Use a fórmula: HTML + CSS = Frontend

Quer ir além e não ser só "revista"? Quer um aplicativo em formato de site? Você vai precisar de JavaScript. Não tem como fugir. Mas não corra. Abrace a dificuldade. Se é difícil, poucos fazem. Se poucos fazem e tem demanda, o dinheiro vai pra quem sabe.

Não vou te enrolar. Em 2026, mais de 75% dos devs usam VS Code. Recomendar outro seria loucura.

Desmembrando código

<!DOCTYPE> Aviso na porta: "Aqui é documento HTML 5". Não apague isso. Sem isso o navegador entra em "modo estranho".

<html lang="en"> Língua do site. Coloca lang="pt-br". Se apagar isso seu navegador nem identifica em que língua está.

<head> Cérebro do site. Usuário não vê essa parte. Não deve ser apagado por nada. Ele é uma das bases.

<meta charset="UTF-8"> Tradutor de acentos. Só apague caso não vá usar ç e os acentos.

<meta name="viewport" content="width=device-width, initial-scale=1.0"> Faz funcionar no celular. Não apague, mesmo que a ideia seja funcionar só no pc.

<title<Document>/title> Nome da aba. O que aparece lá em cima no Chrome. Não apague.

<body> Ele é a base de tudo. Todo o conteúdo vai vir aqui. Tudo que o usuário vê estará aqui. Não apague.

Por fim, tudo que é aberto, deve ser fechado. 90% das coisas ficaram dentro do html e nele algumas ficaram dentro do head, enquanto no body 95% das coisas que for colocar estarão nele.

Primeiro "Hello World"

Tudo que é aberto é fechado automaticamente. Se apertar "p" e enter o VsCode entende que você está tentando puxar a função <p> e finaliza com </p>.

A função "p" seria o print do html. Se quiser pode começar com o básico. Coloque um <p>Hello World</p> dentro do seu <body>. Mas a real é que você não precisa do "p" para fazer isso. Ele serve mais para você declarar para os servidores que isso tudo que você está escrevendo dentro do "p" é texto.

Faça questão de declarar o que é cada coisa. Isso vai te ajudar a configurar seu site depois. Se algo é texto, use o "p" e se algo é título use o "h1", "h2" e etc, olha aí mais um código pra testar.

Na verdade, vou te mostrar na prática, vem comigo:

  • O título dessa página, aquele "HTML: A estr..." ele é o título principal e por isso ele é o "h1" da página.
  • Os outros títulos como o "O que é HTML" são "h2", eles ainda são títulos, mas não são os principais e por isso são "h2" e não "h1".
  • Todos os textos dessa página estão em "p".
