HTML: A estrutura do site

Esse é o nível intermediário de HTML. Até agora, você viu apenas o básico da estrutura.

No entanto, um bom site precisa ser organizado. Por isso, recomendo que você só comece a Introdução ao CSS depois de ter uma base sólida em HTML. Caso contrário, isso vai prejudicar seu progresso.

Um site desorganizado só dá dor de cabeça. Para ter ideia: uma página média deste site tem 300 linhas de código. Agora imagine várias páginas com 300 linhas cada. Sem organização, tudo vira uma bagunça. E por não dedicar tempo à organização agora, você vai gastar muito mais tempo depois só para configurar o site.

Por que organizar? Google

O Google é um robô cego lendo teu site com Ctrl+F. Ele não vê cor, não vê bonito. Ele vê hierarquia e significado. Você pode enfeitar o seu site o quanto quiser pelo CSS que no final o Google vai querer saber se o seu HTML está oganizado ou não.

Google não é teu amigo. É um robô com meta de entregar o melhor resultado em 0.3s. Caso cumpra com o que ele quer, ele vai te recompensar de três formas:

  • Posição na busca: HTML semântico é um dos 200+ fatores de rank. Não é o maior, mas é "fácil" de fazer. <h1> correto vale mais que 100 backlinks lixo.
  • Destaque na busca: Só aparece se teu HTML tiver significado. Isso dá 30% mais cliques. Google te dá destaque porque teu HTML "explicou" o conteúdo pra ele.
  • Quantas páginas o Google visita: Google tem um limite de quanto tempo ele pode ler um site. Se teu HTML é bagunça, ele gasta tempo tentando entender e desiste de páginas profundas. Site Semântico = Google entende rápido e visita 500 páginas/dia.

Por que organizar? CSS

CSS não liga pra significado. CSS liga pra "como eu acho esse elemento pra pintar?". Caso seu site tenha um monte de <div> ao invés de uma organização de verdade. Quem vai se prejudicar tentando organizar o CSS vai ser você.

Claro, você pode colocar um monte de class e id nas suas <div>. Mas ao fazer isso, você vai bater na coluna esquerda: "Google te ignora". O segredo é usar tag semântica pro Google te entender e usar class pra "você" se entender com o CSS.

Está achando que CSS é brincadeira? Mas falando sério. O CSS vai ser o maior causador de problemas do seu site. Nem o JavaScript vai te dar tanta dor de cabeça quanto um CSS em um HTML mal organizado. Quando seu site estiver grande não vai ser o HTML ou JS que vai puxar suas pernas, vai ser o CSS.
